# Macrostructures — named page shapes

A **macrostructure** is the whole-page skeleton: heading placement, section rhythm, how the fold hands off to the scroll, where proof and CTA live, what the divider language is. It is a separate axis from mood (which sets color + type) and from screen type (which sets what content a screen holds). Two landing pages can share a mood and still be structurally different pages — or share the generic hero → 3-feature-cards → testimonial → CTA → footer rhythm and read as the same template in two color-swaps.

That generic rhythm is the single most recognizable "an AI built this" tell at the page level, and it survives a perfect palette. This file exists so structure varies per project the same way `scripts/generate_palette.py` makes color vary per project. **The color engine already prevents color monoculture; this prevents structural monoculture.** Both axes have to move for two projects to actually feel like different sites.

## How to use this file

1. **Pick one macrostructure by name before building** — from the index below, not by reaching for the first shape that comes to mind (that reach is always the generic one). Picking on the page, out loud, is the accountability step that breaks the default-attractor.
2. **Diversify against the last build.** Read `references/diversification.md` and the project's `.tastemaker/log.json` first — your pick must differ from the last macrostructure this project (or this session) produced. State the rotation in one line: *"Last build was Feature Stack; picking Editorial Index this time because the brief is a content-led studio site."*
3. **Then pick the component archetypes** that fill it, from `references/component-catalog.md` — a macrostructure is the page shape; the catalog is the parts inside it.
4. **Record the pick** in `.tastemaker/style-lock.md`'s Structure section and stamp it in the built CSS (see `references/diversification.md`).

Macrostructures below are for **public/marketing pages** (landing, product, about, pricing-as-page) — that's where structural sameness bites hardest. **App shells** (dashboards, internal tools behind a sidebar) have their own governing shape in `references/component-patterns.md`'s App shell section — don't pick a marketing macrostructure for a shell; the sidebar-plus-topbar frame *is* its macrostructure.

## The index

Pick one. Each line is name — one-line shape — when to reach for it — mood affinity (not a hard restriction; any mood can wear any shape if the brief wants it).

- **1 · Feature Stack** — Hero, then features revealed one full-width band at a time down the scroll, each an alternating text/visual split. The workhorse product page — but built as *bands*, not a 3-up card grid, and every band leads with a real mockup/chart, not a paragraph. *When:* a product with 3-6 distinct capabilities that each deserve a real visual. *Mood:* any; the default reach for premium/technical SaaS.
- **2 · Bento Showcase** — An asymmetric grid of mixed-span tiles (1×1, 2×1, 2×2) where each tile shows one capability as a small live-looking UI fragment. Visual rhythm comes from tile size, not repetition. *When:* a product with many small features that are more compelling shown-at-a-glance than explained in sequence. *Mood:* premium, technical, playful.
- **3 · Editorial Index** — Reads like a well-set magazine or a studio's index page: a masthead, a numbered or categorized list of work/sections, generous type, hairline rules. Content-led, not conversion-led. *When:* portfolios, agencies, publications, "this is who we are" pages. *Mood:* elegant, premium.
- **4 · Long-Scroll Narrative** — A single argument told top to bottom: problem → turn → solution → proof → close, each beat a full section with a scroll-linked reveal. The page is a story, not a menu. *When:* a non-obvious product that needs explaining before the pitch lands; a manifesto; a launch. *Mood:* any; strong for warm, elegant, technical.
- **5 · Stat-Led** — A single dominant number or a tight row of real metrics is the hero and the spine of the page; everything else supports the figures. *When:* the product's story is genuinely quantitative and the numbers are **real** (never invented — see the honesty rule below). *Mood:* premium, technical.
- **6 · Gallery Grid** — The work is the pitch: a grid of real photography, product shots, or case tiles fills the page, chrome kept minimal. *When:* commerce, photography, food, travel, physical product, anything where the imagery sells. *Mood:* elegant, warm, playful.
- **7 · Product Demo / Workbench** — The hero is the product in use (a real annotated mockup, an interactive-looking capture, a clipped-by-viewport demo), and the page is organized around *doing*, not describing. *When:* dev tools, apps, anything where seeing it work is the strongest argument. *Mood:* technical, premium.
- **8 · Split Diptych** — The page runs as a two-column composition — a persistent statement/nav column beside a scrolling content column, or a hard left/right split fold. *When:* studios, single-voice products, anything wanting an art-directed rather than stacked feel. *Mood:* elegant, premium, playful.
- **9 · Conversational FAQ** — The page is built around real questions the audience actually asks, answered directly, with proof and CTA folded into the answers. *When:* products fighting a specific objection or a trust gap (regulated, high-consideration, "is this legit"). *Mood:* warm, technical.
- **10 · Manifesto** — Type-forward, few images, one strong point of view carried by big statements and short lines. The restraint *is* the design. *When:* brand-led launches, opinionated products, a studio staking a position. *Mood:* elegant, premium, technical.
- **11 · Catalogue** — A structured, almost tabular listing (products, plans, specs, releases) presented with editorial care rather than as a bare table. *When:* commerce with many SKUs, pricing-as-page, changelogs, spec-heavy products. *Mood:* technical, elegant.
- **12 · Poster Fold** — One full-bleed art-directed fold (a photograph, a hand-built illustration, a bold type composition) carries the whole first screen; the page below becomes something quieter. *When:* the brand or a single image *is* the message — events, fashion, a hero launch moment. *Mood:* elegant, playful, warm.

## Choosing well

- **Match shape to argument, not to habit.** The Feature Stack is the right answer surprisingly often *and* the wrong answer to reach for reflexively — the whole point is that the pick is deliberate. If the brief's strongest argument is "look how much people use this," that's Stat-Led; "look at the work," Gallery Grid; "let me explain why this matters," Long-Scroll Narrative. Pick the shape that carries the actual argument.
- **Vague brief → offer, don't default.** If there's genuinely no signal (no tone, no clear argument), offer the user three macrostructures from categorically different groups (one grid-led like Bento, one document-led like Long-Scroll, one poster-led like Manifesto) — three concrete choices, not a survey of all twelve.
- **Honesty rule (binds every macrostructure).** Stat-Led, Catalogue, and any proof band must use **real** numbers, logos, and testimonials, or an honest placeholder (`—` plus a labelled "metric to confirm" block) — never an invented "+47% conversion" / "trusted by 50,000+ teams" to fill the shape. A fabricated stat is slop the moment it's invented, and it's the fastest way a proof-led page loses trust. If the user supplied no real figures, pick a shape that doesn't demand them, or leave the slot honestly empty.
- **The macrostructure picks most of the structure; you still choose the components.** It sets heading placement, section rhythm, divider language, and reveal. You choose the nav, hero, feature, proof, CTA, and footer archetypes from `references/component-catalog.md` to fill it — and those diversify too.
